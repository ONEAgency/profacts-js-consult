Profacts-js-consult

A javascript modal pop-up implementation for use in external websites that accepts settings

View on GitHub Download .zip View on Npm

Profacts Modal Pupup

The profacts modal popup is a javascript component in the form of a npm package that accepts an ammount of parameters so that the modal window will react in different ways. The package can be downloaded with npm. 

npm install profacts-js-consult

Demo

You can view a basic demo here. The settings that were used to generate the popup on the demo page can be found below. 

<script id="profacts-modal" type="text/javascript" src="dist/js/modal.min.js?campaignkey=profactscampaign
&startdate=2015-11-05T09:00:00&enddate=2015-11-06T09:43:30&showratio=100&expireratio=1h
&templategroupname=profactsmodaltemplates&templatename=popup_1"></script>
Requirements: The script must have the id profacts-modal becaus this is used to get the query params. The script must be included at the bottom of the page in the body tag.

Documentation

As you can see the script can take a number of settings that can be used to configure the behaviour of the modal popup window. Let's look at the complete list of settings that can be used.

Parameter list:

campaignkey: defaults to profactscampaign

This key acts as a unique identifier for setting the cookies of the modal window, when a campaign ends and another popup is provided this key must be changed in order to avoid cookie collisions.

startdate: defaults to 2015-01-01T0:00:01

The modal popup will be shown from this date on until the enddate parameter

enddate: defaults to 2016-01-01T0:00:01

The modal popup will be shown until this date.

Info: The enddate parameter is also used to set the accepted cookies expire date for when the user has clicked the accept button.

showratio: defaults to 100

This is the ratio at which the popup is shown, if this is 50 for example the pop up can be shown at the chance of 1/2.

This setting can be used in conjunction with the expireratio setting, but beware when you combine them the behavior can be unexpected.

But normaly you would never use these 2 combined, it's either based on the expireratio or based on the showratio.

Thats is why the defaults of these two are set properly.

expireratio: defaults to 0

This is the ratio at which the popup is shown based on a cookie setting, when the popup has opened this cookie is set if it is not 0.

The expire date of the cookie can be set in hours or in minutes by providing the suffixes m or h

Info: So 2h means that the pupup will reappear after 2 hours unless the user has clicked the accept button naturally.
2m means that the pupup will reappear after 2 minutes unless the user has clicked the accept button naturally.

templategroupname: defaults to profactsmodaltemplates

This is an extra setting for the templates, you can use this setting to create more templates and bind them to a different key.

Info: This setting is not that important and can be pretty much ignored.

templatename: defaults to popup_1

This key is used to get the contents of the popup modal that is compiled into dist/js/templates.js.

This is the name of the html file that you can find in the src/templates folder, say for example you want to make a different popup with other content. Then you add your template in that directory run gulp js, and you can access the template by changing this key to the name of your template. Then you will have your new content in your popup.

Info: But beware, there are 2 things that are needed internally by the code, first one is the buttons with the classes button-accept and button-decline. And the second one is the overlay div with it's class overlay. The rest is customizable.

About the Styling (scss):

Everything is compiled with gulp into the css folder, this can be altered and you can create extra files that you can use in your new templates. For example: you can duplicate the template-styles.scss file and rename it to template2.scss and then change the @import "layout/template"; line to another file with the scss for the second popup. Or you can link up a css file that is on a cdn in your template, it is really up to you.
